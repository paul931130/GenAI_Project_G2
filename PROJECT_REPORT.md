# 詩境｜生成式 AI 古典詩詞互動系統 專案報告

## 一、專案主題

「詩境」是一個以古典詩詞為核心的互動式網頁系統。使用者可以輸入作者、篇名、詩句片段或意境描述，系統會先從本機 `poems.json` 詩庫查找作品；若本機沒有命中，則可使用 Gemini API 生成詩詞解讀內容。作品目標不是單純查詩，而是讓古典詩詞以「詩魂現身」的方式被重新閱讀。

## 二、問題意識

古典詩詞常見的學習方式偏向背誦、註釋與考題整理，容易讓讀者只看見字面意思，卻忽略詩人的情緒、時代背景與作品的美感。本專案想解決三個問題：

1. 查詩工具通常缺少互動性，使用者不容易進一步追問。
2. AI 生成內容可能不穩定，因此需要本機詩庫作為可控資料來源。
3. 傳統詩詞介面多半偏資料表或文章頁，不容易呈現詩詞本身的氣氛。

## 三、系統功能

### 1. 通靈問詩

使用者輸入詩名、作者、詩句或意境後，系統會顯示詩名、作者朝代、原文、白話釋義、情緒分析、創作背景與來源標籤。來源會明確標示為「Gemini」或「本機詩庫」。

### 2. 本機詩庫備援

本機資料使用 `poems.json`，目前主要收錄唐詩。當沒有 API key、API 配額受限或 Gemini 回應不穩定時，系統仍可查詢本機詩庫中的作品。

### 3. 詩魂問答

使用者可針對目前顯示的作品繼續提問。若 Gemini API 可用，會以詩人第一人稱回應；若 API 不可用，則改用本機詩作資料產生簡短回覆。

### 4. 雙魂對話

使用者可選擇兩位詩人與一個主題，讓系統生成兩位詩人之間的對話。此功能展示 AI 角色設定與古典文學人物語氣模擬。

### 5. 詩藏收藏

使用者可收藏查詢過的詩作，資料存在瀏覽器 `localStorage`，方便在同一台裝置上再次閱讀。

## 四、技術架構

- 前端：HTML、CSS、JavaScript
- 部署：GitHub Pages 靜態網站
- 本機資料：`poems.json`
- 生成式 AI：Gemini 2.5 Flash
- API Key 安全：只暫存在 `sessionStorage`，關閉分頁後清除
- 程式結構：`index.html`、`style.css`、`app.js` 分離，方便維護與快取

## 五、實作特色

1. **本機優先**：先查 `poems.json`，減少 API 消耗。
2. **來源透明**：每次顯示結果都標示來源，避免使用者混淆 AI 與本機資料。
3. **錯誤可操作**：API 失敗時提供重試、切換 API Key、推薦本機類似文章等下一步。
4. **資安考量**：不把 API Key 寫死在程式碼，也不保存到長期 localStorage。
5. **古典 UI**：使用直排詩文、紙色背景、朱紅收藏狀態與古銅色線條建立詩詞閱讀氛圍。

## 六、成果展示方式

展示時可依序操作：

1. 開啟 GitHub Pages 網址。
2. 不輸入 API Key，查詢「靜夜思」或「石壕吏」，展示本機詩庫功能。
3. 貼上 Gemini API Key，展示自動測試與 API 狀態。
4. 查詢「邊塞悲壯」，展示 Gemini 或本機推薦。
5. 收藏一首詩，再切到「詩藏」確認收藏。
6. 開啟「雙魂對話」，讓李白與杜甫談「月亮」或「離別」。

## 七、限制與改進方向

目前本機詩庫主要以唐詩為主，宋詞與其他朝代作品不一定能在本機命中。未來可以擴充詩庫，加入朝代篩選、全文搜尋、詩人頁面、學習進度紀錄與更多可視化分析。

API 方面，前端直連 Gemini 適合課程展示與個人使用；若要公開給大量使用者，應改成後端代理，避免 API Key 暴露與配額被濫用。

## 八、學習反思

本專案讓我練習如何把生成式 AI 與傳統資料庫結合，而不是完全依賴 AI 回答。過程中也理解到 UI/UX、錯誤處理、API key 安全與部署結構都會影響作品是否能真正被使用。最後的成果是一個可部署、可互動、也能在 API 不可用時保留基本功能的古典詩詞系統。
