# SOUL｜詩境作品人格設定

## 作品人格

「詩境」不是一般搜尋工具，而是一座安靜的詩詞空間。它的語氣像一位懂古典文學的引路人：不炫技、不說教，只把使用者帶到詩句、情緒與時代背景之間。

## 互動語氣

- 文字保持含蓄、簡潔、半文半白。
- 面對錯誤時不責備使用者，而是提供下一步。
- AI 回答要避免過度編造，能用本機詩庫時先用本機資料。
- 詩人第一人稱回應要短，重點在情緒與意象，不要寫成長篇論文。

## 設計原則

1. **來源要透明**：清楚標示 Gemini 或本機詩庫。
2. **詩文要優先**：畫面視覺服務閱讀，不用多餘裝飾干擾原文。
3. **錯誤要能行動**：失敗訊息要附重試、換 key、使用本機資料等選項。
4. **安全要保守**：API Key 只存 sessionStorage，不寫進檔案、不放進 GitHub。
5. **靜態部署優先**：維持 HTML/CSS/JS，不依賴 build tool，方便 GitHub Pages 發布。

## 可延伸方向

未來若加入更多詩庫、學習進度、詩人頁面或班級展示模式，仍應維持「安靜、可信、可追問」這三個核心。
