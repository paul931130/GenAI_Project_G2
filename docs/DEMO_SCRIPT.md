# 詩境 Demo 影片腳本（約 5 分鐘）

## 0:00–0:30 開場

旁白：

> 大家好，這是我的生成式 AI 古典詩詞互動系統「詩境」。它不是單純查詩，而是把本機詩庫、Gemini API、詩魂問答與古典風格介面整合在一起，讓使用者用作者、詩名、詩句或意境召喚詩作。

操作：

- 開啟網站首頁
- 簡單展示「通靈問詩」主畫面

## 0:30–1:20 本機詩庫查詢

旁白：

> 這個系統的第一個重點是本機詩庫優先。即使不輸入 Gemini API Key，也可以直接查詢 poems.json 裡的唐詩資料。

操作：

1. 不輸入 API Key
2. 輸入 `靜夜思`
3. 按「焚香問詩」
4. 展示詩名、作者、原文、來源標籤與分析內容

補充旁白：

> 這裡可以看到來源標示為本機詩庫，因此沒有消耗 Gemini API 額度。

## 1:20–2:00 隨機召喚

旁白：

> 第二階段新增了「隨機召喚」功能。使用者不需要輸入文字，只要按「隨機一首」，系統就會從本機 poems.json 隨機抽出一首詩。

操作：

1. 回到輸入區
2. 按「🎲 隨機一首」
3. 展示隨機出現的詩作

補充旁白：

> 這個功能適合展示時快速產生案例，也可以讓使用者探索自己原本不熟悉的作品。

## 2:00–2:50 免費瀏覽器朗讀

旁白：

> 第三個新增功能是免費朗讀。這裡沒有使用 ElevenLabs，也不需要語音 API key，而是使用瀏覽器內建的 speechSynthesis。

操作：

1. 按詩文原文旁的「朗讀」
2. 等待朗讀開始
3. 再按一次停止
4. 按詩魂自語區的「朗讀」

補充旁白：

> 這樣即使在沒有 API key 的情況下，使用者仍然可以完成查詩、看詩與聽詩的完整體驗。

## 2:50–3:30 詩藏收藏

旁白：

> 使用者也可以收藏喜歡的詩。這次改版修正了原本有兩個「詩藏」按鈕的問題，現在只保留主分頁中的詩藏入口，並直接顯示收藏數量。

操作：

1. 按「收藏」
2. 看左側「詩藏」數字變化
3. 切換到「詩藏」頁面
4. 點收藏卡片回到詩作

## 3:30–4:20 Gemini API 進階功能

旁白：

> 如果使用者需要查詢本機詩庫沒有收錄的作品，或想用更抽象的意境召喚詩詞，可以貼上 Gemini API Key。貼上後系統會自動測試一次，但會避免重複測試與過度請求。

操作：

1. 點 `API KEY`
2. 說明 key 只存在 sessionStorage
3. 可示範輸入抽象意境，例如 `邊塞悲壯`
4. 展示 Gemini 或本機推薦結果

補充旁白：

> API 不是基本功能的必要條件，而是進階分析與問答能力的補充。

## 4:20–5:00 雙魂對話與總結

旁白：

> 最後是雙魂對話。使用者可以選擇兩位詩人與主題，讓系統模擬詩人之間的對話。

操作：

1. 切到「雙魂對話」
2. 選李白與杜甫
3. 主題輸入 `月亮`
4. 按「召喚兩位詩魂」

結尾旁白：

> 總結來說，詩境結合了本機資料、生成式 AI、收藏、朗讀與互動式介面。第二階段改版後，即使沒有 API key，也可以使用本機詩庫、隨機召喚與免費朗讀，讓作品更穩定，也更適合展示與實際使用。
