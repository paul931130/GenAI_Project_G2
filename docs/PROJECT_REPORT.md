# 詩境｜生成式 AI 古典詩詞互動系統 專案報告

## 一、專案主題

「詩境」是一個以古典詩詞為核心的互動式網頁系統。使用者可以輸入作者、篇名、詩句片段或意境描述，系統會先從本機 `poems.json` 詩庫查找作品；若本機沒有命中，則可使用 Gemini API 生成詩詞解讀內容。作品目標不是單純查詩，而是讓古典詩詞以「詩魂現身」的方式被重新閱讀。

## 二、問題意識

古典詩詞常見的學習方式偏向背誦、註釋與考題整理，容易讓讀者只看見字面意思，卻忽略詩人的情緒、時代背景與作品的美感。本專案想解決三個問題：

1. 查詩工具通常缺少互動性，使用者不容易進一步追問。
2. AI 生成內容可能不穩定，因此需要本機詩庫作為可控資料來源。
3. 傳統詩詞介面多半偏資料表或文章頁，不容易呈現詩詞本身的氣氛。

## 三、第二階段改版內容

### 1. 修正重複詩藏按鈕

原本介面同時有主分頁「詩藏」與獨立收藏按鈕，手機或窄畫面下容易造成操作混亂。本次改版移除獨立「詩藏」按鈕，將收藏數量 badge 整合到主分頁中的「詩藏」。

### 2. 新增隨機召喚

在「焚香問詩」下方新增「🎲 隨機一首」按鈕。此功能直接從本機 `poems.json` 抽取作品，不會呼叫 Gemini API，適合展示、探索與避免 API 配額消耗。

### 3. 新增免費瀏覽器朗讀

詩作原文與詩魂自語區新增「朗讀」按鈕。朗讀使用瀏覽器內建 Web Speech API 的 `speechSynthesis`，不需要 ElevenLabs、不需要後端伺服器，也不消耗額度。

### 4. 文件同步更新

同步更新 README、使用者指南、專題報告，並新增 demo 腳本，讓 GitHub 頁面、操作文件與展示影片流程保持一致。

## 四、系統功能

### 1. 通靈問詩

使用者輸入詩名、作者、詩句或意境後，系統會顯示詩名、作者朝代、原文、白話釋義、情緒分析、創作背景與來源標籤。來源會明確標示為「Gemini」或「本機詩庫」。

### 2. 本機詩庫備援

本機資料使用 `poems.json`，目前主要收錄唐詩。當沒有 API key、API 配額受限或 Gemini 回應不穩定時，系統仍可查詢本機詩庫中的作品。

### 3. 隨機召喚

使用者可以不輸入任何文字，直接按「🎲 隨機一首」探索本機詩庫。這降低了初次使用門檻，也方便展示時快速產生案例。

### 4. 免費朗讀

使用者可以朗讀詩文原文與詩魂自語。此功能不依賴外部 TTS 服務，適合 GitHub Pages 純靜態部署。

### 5. 詩魂問答

使用者可針對目前顯示的作品繼續提問。若 Gemini API 可用，會以詩人第一人稱回應；若 API 不可用，則改用本機詩作資料產生簡短回覆。

### 6. 雙魂對話

使用者可選擇兩位詩人與一個主題，讓系統生成兩位詩人之間的對話。此功能展示 AI 角色設定與古典文學人物語氣模擬。

### 7. 詩藏收藏

使用者可收藏查詢過的詩作，資料存在瀏覽器 `localStorage`，方便在同一台裝置上再次閱讀。

## 五、技術架構

| 模組 | 技術 | 用途 |
|---|---|---|
| 前端頁面 | HTML | 頁面結構與語意標記 |
| 視覺介面 | CSS | 古典紙本風格、直排詩文、響應式設計 |
| 互動邏輯 | JavaScript | 搜尋、隨機召喚、收藏、問答、朗讀 |
| 本機資料 | poems.json | 可控詩詞資料來源 |
| 生成式 AI | Gemini 2.5 Flash | 意境式查詢、詩魂問答、雙魂對話 |
| 免費 TTS | Web Speech API | 瀏覽器內建朗讀 |
| 部署 | GitHub Pages | 純靜態網站發布 |

## 六、實作特色

1. **本機優先**：先查 `poems.json`，減少 API 消耗。
2. **來源透明**：每次顯示結果都標示來源，避免使用者混淆 AI 與本機資料。
3. **低門檻體驗**：不輸入 API key 也能查唐詩、隨機召喚與朗讀。
4. **錯誤可操作**：API 失敗時提供重試、切換 API Key、推薦本機類似文章等下一步。
5. **受控 API Key 測試**：Gemini key 貼上後會延遲輕量測試一次，並用 debounce、同 key 去重與 429 冷卻避免 Too many requests。
6. **資安考量**：不把 API Key 寫死在程式碼，也不保存到長期 localStorage。
6. **古典 UI**：使用直排詩文、紙色背景、朱紅收藏狀態與古銅色線條建立詩詞閱讀氛圍。

## 七、成果展示方式

展示時可依序操作：

1. 開啟 GitHub Pages 或本機 localhost 網址。
2. 不輸入 API Key，查詢「靜夜思」或「石壕吏」，展示本機詩庫功能。
3. 按「🎲 隨機一首」，展示不消耗 API 的本機探索功能。
4. 按詩文旁的「朗讀」，展示免費瀏覽器朗讀。
5. 收藏一首詩，再切到「詩藏」確認收藏。
6. 若需要 AI 功能，再貼上 Gemini API Key；系統會先自動輕量測試 key，通過後再查詢抽象意境或進行詩魂問答。
7. 開啟「雙魂對話」，讓李白與杜甫談「月亮」或「離別」。

## 八、限制與改進方向

目前本機詩庫主要以唐詩為主，宋詞與其他朝代作品不一定能在本機命中。未來可以擴充詩庫，加入朝代篩選、全文搜尋、詩人頁面、學習進度紀錄與更多可視化分析。

瀏覽器朗讀的音色取決於使用者裝置，品質不如專業 TTS 服務。未來若需要更自然的真人語音，可再加入 ElevenLabs 或其他 TTS 作為進階選項。

API 方面，前端直連 Gemini 適合課程展示與個人使用；若要公開給大量使用者，應改成後端代理，避免 API Key 暴露與配額被濫用。

## 九、學習反思

本專案讓我練習如何把生成式 AI 與傳統資料庫結合，而不是完全依賴 AI 回答。第二階段改版進一步讓系統在沒有 API key 的情況下仍能完成查詩、隨機探索與朗讀，提升作品的穩定性與展示完整度。過程中也理解到 UI/UX、錯誤處理、API key 安全與部署結構都會影響作品是否能真正被使用。
