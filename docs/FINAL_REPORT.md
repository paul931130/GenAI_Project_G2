# FINAL_REPORT｜詩境期末專題報告

> 專題名稱：詩境——生成式 AI 古典詩詞互動系統  
> 版本：v1.4.2  
> 類型：生成式 AI 應用、古典詩詞檢索、互動式前端網站

## 一、專題摘要

「詩境」是一個以古典詩詞為核心的生成式 AI 互動系統。專案目標不是單純建立詩詞資料庫，而是將古典詩詞、前端互動、資料清理與生成式 AI 串接成一個可展示、可閱讀、可互動的作品。

使用者可輸入詩名、作者、詩句片段或意境描述，系統會優先查詢本機 `poems.json` 詩庫；若 Gemini API 可用，則進一步產生延伸分析與詩魂問答。本專案採純前端架構，可部署於 GitHub Pages，並透過本機詩庫作為 API 失效時的備援機制。

## 二、研究動機

古典詩詞的學習常遇到兩個問題。第一，資料雖然豐富，但原文、譯文、注釋、賞析與背景常分散在不同來源，閱讀體驗不連續。第二，生成式 AI 雖能協助理解文本，但若沒有穩定的資料結構與錯誤處理，輸出容易混亂，甚至將註釋、異文或英文翻譯誤當成正文。

因此，本專案希望建立一個具備以下特色的系統：

1. 以古典美學呈現詩詞文本。
2. 以本機詩庫確保基本功能穩定。
3. 以 Gemini API 提供延伸解讀與互動。
4. 透過資料清理與前端格式控制，提高輸出品質。
5. 在 API 受限時仍可完成展示流程。

## 四、主要功能

### 1. 本機詩庫查詢

系統內建 315 首唐詩資料，可支援詩名、作者與詩句片段查詢。即使沒有 API Key，仍可完成核心展示。

### 2. 詩魂現身

搜尋後會產生詩作閱讀卡，包含篇名、作者、原詩、白話釋義、情緒意象與背景。介面採古典紙面與直排排版設計，使作品具有「詩卷」感。

### 3. 校注與正文分離

部分資料中的異文註記會直接附在詩句後方，例如 `(唯 通：惟)` 或 `(明年 一作：年年)`。若直接顯示在原詩區，會破壞直排版面。系統會自動將這類資料抽出，放入校注區。

### 4. 詩庫輸出

原始資料中常包含 `譯文及注釋`、`注釋`、`註解`、`英譯` 等標籤。系統會先清理這些標籤，再將內容呈現於白話釋義與背景區，避免輸出像原始爬蟲資料。

### 5. 長詩直排白框伸縮

長篇作品若使用固定高度直排，會形成過多直欄並降低可讀性。v1.4.1 起，原詩白色方框會依最長詩句自動垂直加高，保留直排展示感，也提升長篇詩閱讀體驗。

### 6. API 狀態與備援

系統支援 Gemini API Key 測試與狀態顯示。當 API 配額受限或未輸入 Key 時，本機詩庫仍可提供基本功能，避免展示中斷。

### 7. 收藏、複製與朗讀

收藏功能使用 localStorage，朗讀功能使用 Web Speech API，複製功能可輸出整理後的詩作內容。這些功能皆不依賴後端伺服器。

## 六、學習反思

本專案的重點不只是接上生成式 AI，而是理解一個可展示作品需要完整的資料、介面與錯誤處理流程。開發過程中，最常見的問題不是模型不會回答，而是資料來源不乾淨、瀏覽器快取、手機排版、API 配額限制與使用者流程不穩定。

透過多次版本修正，本專案逐步從「能跑」調整為「能穩定展示」。這也反映生成式 AI 應用的核心能力：不只是呼叫 API，而是將模型能力包裝在穩定、可理解、可維護的系統之中。
