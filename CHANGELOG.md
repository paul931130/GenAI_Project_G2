## v1.5.0｜正式發布版：詩庫輸出清理、長序文分離與文件整合

1. **補強譯文輸出清理**

   - 原詩區只保留真正詩句，避免題序進入直排原文區。

3. **補齊發布說明文件**
   - 更新 `README.md`、`USER_GUIDE.md`、`CHANGELOG.md`、`docs/FINAL_REPORT.md`、`docs/DEMO_VIDEO_SCRIPT.md` 與 `docs/SOUL.md`。
   - 文件結構精簡為期末展示需要的報告、Demo 腳本與設計原則。

4. **檢查項目**
   - `poems.json` 可正常解析，共 315 首。
   - `app.js` 語法檢查通過。
   - `content` 內未發現校注括號、譯文標籤、注釋標籤、英文段落或長篇題序。
   - 重新抽查《怨情》《黃鶴樓送孟浩然之廣陵》《琵琶行》《次北固山下》《夕次盱眙縣》等作品輸出。

## v1.4.1｜本機詩庫輸出清理

### 更新重點

1. **修正「詩魂自述」誤顯示原始資料標籤**
   - 移除本機詩庫輸出中的 `譯文及注釋`、`譯文`、`直譯`、`韻譯`、`意譯`、`漢譯` 等標題。
   - 白話釋義區只保留真正的白話內容，不再顯示資料來源標籤。

2. **修正「魂歸何處」誤顯示英譯資料**
   - 避免 `others` 欄位中的 `英譯` 被當成創作背景顯示。
   - 若沒有可用創作背景，改用中文賞析作為備援，不再輸出英文翻譯。

3. **強化本機詩庫清理規則**
   - 新增中文優先選取邏輯。
   - 自動略過純英文或英文翻譯段落。
   - 支援切開混合型資料，例如同一段內同時含有漢譯與英譯時，只取漢譯部分。

4. **同步清理 poems.json**
   - 移除純英文翻譯項目。
   - 檢查 `content` 內沒有殘留會破壞原詩顯示的異文註記。

### 測試重點

建議測試：

- `怨情`：確認「詩魂自述」只顯示中文白話，不顯示 `譯文及注釋`。

- `鹿柴`、`哥舒歌`、`次北固山下`、`夕次盱眙縣`：確認混合翻譯資料只取中文段落。

## v1.4.0｜校注分離與快取修正

### 更新重點

1. **修正原詩正文混入校勘註記的問題**
   - 舊版可能顯示：`孤帆遠影碧空盡，唯見長江天際流。 (唯 通：惟)`
   - 新版原詩只顯示：`孤帆遠影碧空盡，唯見長江天際流。`
   - 校注區另顯示：`唯，通「惟」。`

2. **新增「校注／異文」自動分離機制**
   - 支援 `(通：)`、`(一作：)`、`(又作：)`、`(版本...)` 等常見格式。
   - 原詩直排區只保留真正詩句。
   - 即使 `poems.json` 仍殘留少量異文括號，前端也會自動拆分，避免直接破壞版面。

3. **修正 Gemini 輸出格式**
   - Prompt 要求 Gemini 將校勘文字放入 `variantNotes`。
   - `fullText` 只允許放原詩正文，不允許混入翻譯、註釋、英文、拼音、賞析或版本說明。

4. **修正手機版直排穩定性**
   - 校注改為橫排、小字、低權重顯示。
   - 避免括號註記或長註解撐開詩文白框。

5. **修正本機詩庫快取問題**
   - `poems.json` 讀取時自動加入版本參數與時間戳。
   - 使用者不需要手動在網址後面加 `?v=`。
   - 降低手機或瀏覽器持續讀到舊 `poems.json` 的機率。

6. **更新前端資源版本**
   - `index.html` 已更新 `app.js` 與 `style.css` 的版本參數。
   - 降低 GitHub Pages 或瀏覽器載入舊版資源的機率。

### 發布前測試

建議至少測試：

- 搜尋「黃鶴樓送孟浩然之廣陵」
- 搜尋「故人西辭黃鶴樓」
- 搜尋含異文的作品，例如「送別 / 山中送別」、「怨情」、「八陣圖」
- 重新開啟手機版頁面，確認沒有讀到舊詩庫
- 隨機召喚、收藏、複製、朗讀是否正常

## v1.3.0｜本機相近作品與錯誤提示修正

### 更新重點

1. **改善本機相近作品推薦**
   - 當本機詩庫查不到完全命中的作品時，系統會嘗試推薦相近詩詞。
   - 減少直接顯示「本機詩庫未找到相近作品」的情況。

2. **改善錯誤流程**
   - API 不可用時，使用者仍可改用本機查詢、隨機召喚或重新輸入。
   - 失敗狀態不再只停留在錯誤訊息，而是提供可操作的下一步。

3. **維持 API 狀態一致性**
   - 使用本機推薦時，不再誤判為 API 斷線。
   - 來源標籤更清楚區分「Gemini」與「本機詩庫」。

## v1.2.0｜UI、API 狀態與手機版修正

### 更新重點

1. **修正原文白色方框高度**
   - 詩文區依詩句長度自動調整高度。
   - 短詩不再硬撐過高，文字更接近視覺中心。

2. **修正隨機召喚與 API 狀態互相干擾**
   - 隨機召喚來源為本機詩庫，不會呼叫 Gemini API。
   - 按下「隨機一首」後，不再覆蓋原本的 Gemini API 連線狀態。

3. **優化 API Key 測試流程**
   - 貼上 key 後採延遲自動測試。
   - 同一把 key 測試成功後不重複測試。
   - 遇到 429 或配額限制時進入冷卻，避免連續打 API。

4. **改善手機版導覽與操作區**
   - 降低導覽列遮擋標題或擠壓內容的機率。
   - 操作按鈕與詩文區在窄螢幕下更穩定。

5. **修正重複詩藏入口**
   - 移除重複的獨立詩藏按鈕。
   - 收藏數量集中顯示在主分頁中的詩藏入口。

6. **新增免費瀏覽器朗讀**
   - 使用 Web Speech API 的 `speechSynthesis`。
   - 不需要 ElevenLabs、不需要語音 API key、不需要後端伺服器。

## v1.1.0｜專案拆分、本機詩庫與部署文件

### 更新重點

1. **拆分前端檔案結構**
   - 將原本集中在 `index.html` 的樣式與邏輯拆分為：
     - `index.html`
     - `style.css`
     - `app.js`
     - `poems.json`

2. **拆出本機詩庫 `poems.json`**
   - 將大量內嵌詩詞資料移出 HTML。
   - 網頁首次需要詩庫時才 lazy-load，降低首頁負擔。

3. **建立純靜態部署流程**
   - 支援 GitHub Pages。
   - 不需要 build tool、不需要後端。
   - 可用 `python -m http.server` 或 VS Code Live Server 本機測試。

4. **新增使用者文件**
   - `README.md`：專案說明與發布方式。
   - `USER_GUIDE.md`：使用者操作說明。
   - `docs/FINAL_REPORT.md`：期末專題報告內容。
   - `docs/SOUL.md`：作品人格與設計原則。

5. **強化 API Key 安全說明**
   - API Key 不寫入程式碼。
   - Key 僅存於 `sessionStorage`，關閉分頁後清除。
   - 不輸入 key 仍可使用本機唐詩查詢、隨機召喚與朗讀。

---

### 初版功能

1. **通靈問詩**
   - 支援輸入作者、詩名、詩句片段或意境描述。
   - 回傳詩名、作者朝代、原詩、白話釋義、情緒、背景與來源。

2. **詩魂現身**
   - 以詩人第一人稱產生簡短回應。
   - 讓查詩不只是資料查詢，而是帶有角色互動感。

3. **分頁式內容閱讀**
   - 支援通靈對話、賞析、詩藏等主要閱讀區塊。

4. **收藏與複製**
   - 可收藏詩作到瀏覽器 `localStorage`。
   - 可複製格式化後的詩名、作者朝代與原文。

5. **Gemini API 整合**
   - 使用 Gemini 進行 AI 詩詞解析與問答。
   - 初版已具備 API Key 輸入與基礎查詢能力。

## 維護備註

後續如果繼續改版，建議遵守以下規則：

1. 每次功能或資料格式改動，都更新 `CHANGELOG.md`。
2. 如果影響使用者操作，更新 `USER_GUIDE.md`。
3. 如果影響專題展示、技術架構或研究動機，更新 `docs/FINAL_REPORT.md`。
4. 如果改動 `app.js`、`style.css`、`poems.json`，同步更新版本參數，避免 GitHub Pages 快取造成舊版殘留。
5. 發布前至少測試：本機詩庫查詢、Gemini API、隨機召喚、手機版導覽、收藏、朗讀、校注分離。
